## Architecture Overview

### Separation of Concerns

```
Main Menu             Cross-Scene       Race Scene
├─ EventSelector      ├─ EventSession   ├─ RaceManager
├─ SettingsPanel      │  Config         ├─ Athletes
└─ MainMenuController └─ EventSession   └─ Track System
                         Manager
```

### Key Design Decisions

1. **EventSessionManager (Persistent)**
   - Uses DontDestroyOnLoad to bridge scene transitions
   - Static Instance property ensures availability
   - HasConfig flag prevents accidental use of default config
   - Justified for inter-scene communication of race configuration

2. **EventDefinition ScriptableObjects**
   - Non-programmer friendly event configuration
   - Scales to hundreds of events without code changes
   - Supports future field events (Long Jump, Shot Put, etc.)

3. **No Breaking Changes**
   - RaceManager works with or without EventSessionManager
   - Existing gameplay systems completely untouched
   - TrackScene can be loaded directly for testing

4. **Modular Panels**
   - IPanelUI interface for reusable pattern
   - Settings panel easily expands with new options (audio, graphics, etc.)
   - Event selector uses dynamic button creation from EventManager

## Configuration Data Flow

```
Main Menu Selections        EventSessionConfig         RaceManager
├─ Selected Event      →    ├─ SelectedDistance   →    ├─ raceDistance
├─ Lane Mode           →    ├─ LaneMode            →    ├─ playerLane
├─ Fixed Lane (if)     →    ├─ FixedLane           →    │  (calc'd)
└─ Input Mode          →    ├─ InputMode           →    └─ currentInputMode
                            └─ GetPlayerLane()
```

## Extensibility Examples

### Adding a New Setting

Add field to EventSessionConfig:
```csharp
public int AIDifficulty { get; set; }
```

Add UI control to SettingsPanel:
```csharp
[SerializeField] private Dropdown aiDifficultyDropdown;
```

No changes to RaceManager needed - settings cascade through EventSessionConfig.

### Adding New Events

1. Create EventDefinition.asset for the new event
2. Add to EventManager.availableEvents array
3. Done! No code changes needed

### Supporting Field Events

1. Create new RaceDistance enum values if needed
2. Create EventDefinition SOs for field events
3. Extend EventSessionConfig only if field-specific settings exist
4. Add logic to RaceManager only if field event behavior differs
