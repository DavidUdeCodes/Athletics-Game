## EventSessionConfig

Purpose: Data container for menu selections

Public Properties:
  - RaceDistance SelectedDistance { get; set; }
  - LaneSelectionMode LaneMode { get; set; }
  - int FixedLane { get; set; }
  - SprintInputMode InputMode { get; set; }

Public Methods:
  - int GetPlayerLane(int defaultLane = 1)
    Returns actual lane: either FixedLane or random 1-8

  - string ToString()
    Returns formatted config string for logging

Example Usage:
  var config = new EventSessionConfig(
      RaceDistance.Distance100m,
      LaneSelectionMode.Fixed,
      3,
      SprintInputMode.Rhythm
  );

## EventSessionManager

Purpose: Persistent configuration manager across scenes

Static Access:
  - EventSessionManager.Instance
    Auto-creates if not present, returns singleton

Public Properties:
  - EventSessionConfig CurrentConfig { get; }
  - bool HasConfig { get; }
    True if config was explicitly set, false otherwise

Public Methods:
  - void SetEventConfig(RaceDistance, LaneSelectionMode, int, SprintInputMode)
    Creates new config from parameters, sets HasConfig = true

  - void SetEventConfig(EventSessionConfig config)
    Uses provided config, sets HasConfig = true

  - void ClearConfig()
    Resets to default config, sets HasConfig = false

  - void Cleanup()
    Destroys manager (call when returning to menu)

Example Usage:
  EventSessionManager.Instance.SetEventConfig(
      RaceDistance.Distance200m,
      LaneSelectionMode.Random,
      1,
      SprintInputMode.ForceControl
  );
  
  // Later, in RaceManager:
  if (EventSessionManager.Instance.HasConfig)
  {
      var config = EventSessionManager.Instance.CurrentConfig;
      playerLane = config.GetPlayerLane();
  }

## EventDefinition (ScriptableObject)

Purpose: Non-programmer friendly event configuration

Create via: Right-click > Create > Athletics > Event Definition

Public Properties (Read-only):
  - string EventName { get; }
  - RaceDistance Distance { get; }
  - string EventCategory { get; }
  - string Description { get; }

Inspector Fields:
  - Event Name: "100m", "200m", "Long Jump", etc.
  - Distance: Choose from RaceDistance enum
  - Event Category: "Sprint", "Middle Distance", "Field", etc.
  - Description: "100 meter sprint", etc.

## EventManager

Purpose: Centralized event lookup and discovery

Inspector Setup:
  - Available Events: Array of EventDefinition SOs

Public Methods:
  - EventDefinition GetEvent(RaceDistance distance)
    Returns event for given distance, or null

  - IReadOnlyList<EventDefinition> GetAllEvents()
    Returns all available events

  - EventDefinition GetEventByName(string name)
    Searches events by EventName field

Example Usage:
  var eventDef = eventManager.GetEvent(RaceDistance.Distance100m);
  string eventName = eventDef.EventName; // "100m"
  
  foreach (var evt in eventManager.GetAllEvents())
  {
      Debug.Log($"Available: {evt.EventName}");
  }

## MainMenuController

Purpose: Orchestrates menu flow and scene transitions

Inspector Assignments:
  - Event Selector Panel: EventSelectorPanel component
  - Settings Panel: SettingsPanel component
  - Race Scene Name: "TrackScene" (or your scene name)

Public Methods:
  - void OnPlayButtonPressed()
    Collects all selections, creates config, loads race scene

  - void OnEventSelectorButtonPressed()
    Shows EventSelectorPanel

  - void OnSettingsButtonPressed()
    Shows SettingsPanel

Button Event Wiring:
  PlayButton.onClick.AddListener(() => mainMenuController.OnPlayButtonPressed());
  EventSelectorButton.onClick.AddListener(() => mainMenuController.OnEventSelectorButtonPressed());
  SettingsButton.onClick.AddListener(() => mainMenuController.OnSettingsButtonPressed());

## EventSelectorPanel : IPanelUI

Purpose: Dynamic event selection UI

Inspector Assignments:
  - Canvas Group: (auto-found if not assigned)
  - Event Button Container: Transform of button parent
  - Event Button Prefab: Button prefab with TextMeshProUGUI
  - Event Manager: (auto-found if not assigned)

Public Methods:
  - void Initialize()
    Fetches events from EventManager, creates buttons

  - RaceDistance GetSelectedDistance()
    Returns currently selected event distance

  - void SetActive(bool active)
    Shows/hides panel with fade effect

Implementation:
  - Dynamically creates buttons from EventManager.GetAllEvents()
  - Each button click calls SelectEvent(RaceDistance)
  - Selected distance stored in _selectedDistance

## SettingsPanel : IPanelUI

Purpose: Configure race preferences (lane, input mode)

Inspector Assignments:
  - Canvas Group: (auto-found if not assigned)
  - Lane Preference Dropdown: Dropdown component
  - Fixed Lane Dropdown: Dropdown component
  - Fixed Lane Container: GameObject (shown/hidden based on mode)
  - Input Mode Dropdown: Dropdown component

Public Methods:
  - void Initialize()
    Populates all dropdowns with options

  - LaneSelectionMode GetLaneMode()
    Returns Random or Fixed

  - int GetFixedLane()
    Returns selected lane (1-8)

  - SprintInputMode GetInputMode()
    Returns Rhythm or ForceControl

  - void SetActive(bool active)
    Shows/hides panel with fade effect

Dropdown Values:
  - Lane Preference: [0="Random", 1="Fixed"]
  - Fixed Lane: [0="Lane 1", 1="Lane 2", ..., 7="Lane 8"]
  - Input Mode: [0="Rhythm", 1="Force Control"]

## LaneSelectionMode Enum

```csharp
public enum LaneSelectionMode
{
    Random,  // 0: RaceManager picks random lane 1-8
    Fixed    // 1: Use specific lane from FixedLane field
}
```

## RaceManager Integration

Start() method now:
1. Checks if EventSessionManager.Instance.HasConfig
2. If true: Reads SelectedDistance, InputMode, calculates lane
3. If false: Uses inspector defaults
4. Calls SetupRace() and BeginRaceStart()

Backward Compatibility:
- Works with or without menu configuration
- Direct TrackScene load uses inspector defaults
- No changes to existing race logic

## Common Setup Mistakes to Avoid

❌ Forgetting to set EventManager.availableEvents array
   → Result: No event buttons appear

❌ Not assigning EventButtonPrefab
   → Result: NullReferenceException in EventSelectorPanel

❌ Scene name mismatch in MainMenuController
   → Result: Scene fails to load

❌ Missing CanvasGroup on panels
   → Result: Auto-created but better to assign manually

❌ Not wiring Play button to MainMenuController.OnPlayButtonPressed()
   → Result: Nothing happens when play is clicked
