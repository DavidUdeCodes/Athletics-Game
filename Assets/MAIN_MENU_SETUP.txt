Main Menu System - Setup Guide

This document describes how to set up the Main Menu system for the Athletics Game.

## Components Created

### Data Classes & Managers

1. **EventSessionConfig** (Assets/Scripts/Core/EventSessionConfig.cs)
   - Stores player's menu selections (event, lane mode, input mode)
   - Calculates actual lane when Random mode is selected
   - Lightweight, no Unity dependencies

2. **EventSessionManager** (Assets/Scripts/Core/EventSessionManager.cs)
   - Persistent manager (DontDestroyOnLoad)
   - Holds current EventSessionConfig across scene loads
   - Provides static Instance accessor for inter-scene communication

3. **EventDefinition** (Assets/Scripts/Data/EventDefinition.cs)
   - ScriptableObject for track event metadata
   - Fields: EventName, Distance, EventCategory, Description
   - Create these in Assets/Resources/Events/ or wherever you organize SOs

4. **EventManager** (Assets/Scripts/Core/EventManager.cs)
   - Manages available EventDefinition instances
   - Provides lookup by distance or name
   - Place in TrackScene with all EventDefinition references

### UI Components

5. **MainMenuController** (Assets/Scripts/UI/MainMenuController.cs)
   - Orchestrates menu panels (Event Selector, Settings)
   - Handles Play button logic
   - Creates EventSessionConfig and triggers scene load

6. **EventSelectorPanel** (Assets/Scripts/UI/EventSelectorPanel.cs)
   - Displays available events dynamically from EventManager
   - Creates event buttons from a prefab
   - Returns selected distance when queried

7. **SettingsPanel** (Assets/Scripts/UI/SettingsPanel.cs)
   - Lane preference (Random/Fixed) dropdown
   - Fixed lane selector (Lanes 1-8)
   - Sprint input mode (Rhythm/Force Control) dropdown

8. **IPanelUI** (Assets/Scripts/UI/IPanelUI.cs)
   - Interface for menu panels
   - Methods: Initialize(), SetActive(bool)

### Modified Existing Components

9. **RaceManager** (Assets/Scripts/Core/RaceManager.cs) - MODIFIED
   - Start() now checks EventSessionManager for configuration
   - Falls back to inspector defaults if no session config exists
   - Supports testing without main menu

## Main Menu Scene Setup

### Step 1: Create Canvas Structure

In the Main Menu scene, create the following hierarchy:

```
Canvas
├── MainMenuPanel
│   ├── Title (TextMeshProUGUI)
│   ├── PlayButton (Button)
│   ├── EventSelectorButton (Button)
│   ├── SettingsButton (Button)
│   ├── ChangeTrackButton (Button) [placeholder]
│   ├── CustomizeCharacterButton (Button) [placeholder]
│   └── PersonalBestsButton (Button) [placeholder]
├── EventSelectorPanel
│   ├── Title (TextMeshProUGUI)
│   ├── BackButton (Button)
│   └── EventButtonContainer (VerticalLayoutGroup)
│       └── (Event buttons created dynamically)
└── SettingsPanel
    ├── Title (TextMeshProUGUI)
    ├── BackButton (Button)
    ├── LaneSection
    │   ├── Label (TextMeshProUGUI)
    │   ├── LanePreferenceDropdown (Dropdown)
    │   └── FixedLaneContainer
    │       ├── Label (TextMeshProUGUI)
    │       └── FixedLaneDropdown (Dropdown)
    └── InputModeSection
        ├── Label (TextMeshProUGUI)
        └── InputModeDropdown (Dropdown)
```

### Step 2: Create EventDefinition ScriptableObjects

Create three SO instances for the track events:

1. **Assets/Resources/Events/Event_100m.asset**
   - Event Name: "100m"
   - Distance: Distance100m
   - Category: "Sprint"
   - Description: "100 meter sprint"

2. **Assets/Resources/Events/Event_200m.asset**
   - Event Name: "200m"
   - Distance: Distance200m
   - Category: "Sprint"
   - Description: "200 meter sprint"

3. **Assets/Resources/Events/Event_400m.asset**
   - Event Name: "400m"
   - Distance: Distance400m
   - Category: "Middle Distance"
   - Description: "400 meter run"

### Step 3: Set Up EventManager in Main Menu Scene

1. Create an empty GameObject named "EventManager" in the Main Menu scene
2. Add the EventManager component
3. In the inspector, set the "Available Events" array to contain the three EventDefinition SOs you created

### Step 4: Create Event Button Prefab

Create a prefab at **Assets/Prefabs/UI/EventButton.prefab**:

```
EventButton
├── Button (Button component)
│   └── Text (TextMeshProUGUI)
└── Image (for visual feedback)
```

Simple structure - just a button with text and an image for background.

### Step 5: Configure Main Menu Controller

1. Create an empty GameObject named "MainMenuController" in the Main Menu scene
2. Add the MainMenuController component
3. Assign references in inspector:
   - Event Selector Panel: Drag the EventSelectorPanel GameObject
   - Settings Panel: Drag the SettingsPanel GameObject
   - Race Scene Name: "TrackScene" (or whatever your race scene is named)

### Step 6: Configure EventSelectorPanel

1. Add the EventSelectorPanel component to the EventSelectorPanel GameObject
2. Assign references:
   - Canvas Group: Get from the panel's root (auto-found if not assigned)
   - Event Button Container: Drag the "EventButtonContainer" Transform
   - Event Button Prefab: Drag the EventButton prefab you created
   - Event Manager: Will auto-find, but you can assign manually

### Step 7: Configure SettingsPanel

1. Add the SettingsPanel component to the SettingsPanel GameObject
2. Assign references:
   - Canvas Group: Get from the panel's root (auto-found if not assigned)
   - Lane Preference Dropdown: Drag the Dropdown component from LanePreferenceDropdown
   - Fixed Lane Dropdown: Drag the Dropdown component from FixedLaneDropdown
   - Fixed Lane Container: Drag the GameObject "FixedLaneContainer"
   - Input Mode Dropdown: Drag the Dropdown component from InputModeDropdown

### Step 8: Wire Up Button Events

In the Canvas, configure the buttons:

1. **PlayButton**
   - On Click: Invoke MainMenuController.OnPlayButtonPressed()

2. **EventSelectorButton**
   - On Click: Invoke MainMenuController.OnEventSelectorButtonPressed()

3. **SettingsButton**
   - On Click: Invoke MainMenuController.OnSettingsButtonPressed()

4. **BackButtons** (in EventSelectorPanel and SettingsPanel)
   - Back button in EventSelectorPanel: Show MainMenuPanel, hide self
   - Back button in SettingsPanel: Show MainMenuPanel, hide self

### Step 9: Verify Scene Setup

- Ensure both "Main Menu" and "TrackScene" are in Build Settings (File > Build Settings)
- Main Menu should be scene 0 or at least listed before TrackScene
- EventSessionManager will be created automatically if not present

## Data Flow

1. **Main Menu Scene Loads**
   - MainMenuController initializes
   - EventSelectorPanel fetches events from EventManager
   - SettingsPanel sets up dropdowns with defaults

2. **Player Configures Race**
   - Selects event from EventSelectorPanel
   - Adjusts lane preference and input mode in SettingsPanel

3. **Player Clicks Play**
   - MainMenuController collects all selections
   - EventSessionConfig is created
   - EventSessionManager stores config
   - TrackScene is loaded

4. **Race Scene Initializes**
   - RaceManager.Start() checks EventSessionManager
   - Uses EventSessionConfig values
   - Falls back to inspector defaults if no session config

## Extending the System

### Adding New Events

1. Create a new EventDefinition SO with the new event data
2. Add it to EventManager.availableEvents array in the Main Menu scene
3. No code changes needed!

### Adding New Settings

To add new options (e.g., AI Difficulty):

1. Add field to EventSessionConfig class
2. Add UI control to SettingsPanel
3. Add getter method to SettingsPanel
4. Update MainMenuController.OnPlayButtonPressed() to include the new field
5. Update RaceManager.Start() to read the new field

### Field Events

When implementing field events (Long Jump, Shot Put, etc.):

1. Create new RaceDistance enum values if needed
2. Create EventDefinition SOs for field events
3. Extend EventSessionConfig if field events need different config (e.g., attempt number)
4. No menu UI changes needed - all handled by EventDefinition/EventManager

## Testing

To test without the main menu:

1. Remove the session config check in RaceManager.Start()
   OR
2. Don't load the Main Menu scene - directly load TrackScene
   - RaceManager will use inspector defaults

RaceManager is backward compatible and works with or without EventSessionManager.

## Troubleshooting

**"EventManager not found"**
- Ensure EventManager component is in the Main Menu scene
- Ensure EventManager has EventDefinition SOs assigned

**Buttons not responding**
- Verify Button.onClick events are wired up correctly in inspector
- Ensure MainMenuController is assigned to button events

**Scene doesn't load**
- Check that TrackScene is in Build Settings
- Verify scene name matches MainMenuController.raceSceneName

**Settings not applying**
- Ensure EventSessionManager is created (check Hierarchy after first load)
- Verify RaceManager.Start() is being called (check console logs)
