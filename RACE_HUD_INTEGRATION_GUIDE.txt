RACE HUD AND RESULTS SYSTEM - INTEGRATION GUIDE

===============================================
IMPLEMENTED COMPONENTS
===============================================

1. RaceTimer (Assets/Scripts/Core/RaceTimer.cs)
   - Starts when the race officially begins (EnterRunningState)
   - Supports both SS.ff and MM:SS.ff formatting automatically
   - Provides OnTimerUpdated and OnTimerFinished events
   - Properties: ElapsedTime, IsRunning, OfficialFinishTime

2. RaceResult Model (Assets/Scripts/Core/RaceResult.cs)
   - Data model for completed athlete results
   - Fields: Placement, AthleteName, Nationality, FinishTime, IsPlayer
   - Methods: GetFormattedTime(), GetOrdinalPlacement()

3. RaceHUD (Assets/Scripts/UI/RaceHUD.cs)
   - Independent from Results UI
   - Displays: Race Timer, Current Speed
   - Shows only during Running state
   - Easily extensible for future HUD elements

4. RaceFinishFlowManager (Assets/Scripts/Core/RaceFinishFlowManager.cs)
   - Listens to player finish event
   - Shows "FINISHED" message with fade animation
   - Configurable delay before Results (default 3 seconds)
   - Integrates with Results Screen

5. ResultsScreen (Assets/Scripts/UI/ResultsScreen.cs)
   - Displays results dynamically from race data
   - Buttons: Play Again (reload scene), Replay (stub), Main Menu
   - Manages Result Row prefab instantiation

6. ResultRow (Assets/Scripts/UI/ResultRow.cs)
   - Component for individual result display
   - Fields: Placement, Nationality Image, Athlete Name, Finish Time
   - Reusable for multiple athletes

===============================================
ARCHITECT CHANGES TO EXISTING SYSTEMS
===============================================

1. Athlete.cs modifications:
   - Added _raceOfficiallyStarted flag
   - Race timer now only increments after movement begins (EnterRunningState)
   - Reset properly on false starts and race config changes
   - Maintains backward compatibility

2. RaceManager.cs modifications:
   - Added OnPlayerFinished event (fires when player crosses finish line)
   - Integrated RaceTimer system
   - New methods: GetRaceResults(), GetAthleteFinishTime(), GetPlayerFinishTime()
   - Tracks athlete finish times using authoritative RaceTimer
   - Maintains all existing functionality

===============================================
SCENE SETUP INSTRUCTIONS
===============================================

Step 1: Create GameObjects in Scene
--------------------------------
a) Race HUD GameObject:
   - Create UI Panel for race HUD
   - Name it "RaceHUD"
   - Add RaceHUD component
   - Add CanvasGroup component (for fade control)

b) Race Finish Flow GameObject:
   - Create empty GameObject
   - Name it "RaceFinishFlowManager"
   - Add RaceFinishFlowManager component
   - Create a TextMeshProUGUI child for "FINISHED" message
   - Assign the text element in inspector

c) Results Screen GameObject:
   - Create UI Panel for results
   - Name it "ResultsScreen"
   - Add ResultsScreen component
   - Add CanvasGroup component

Step 2: Configure RaceHUD UI Elements
-------------------------------------
In the RaceHUD panel, create:
1. Timer Display (TextMeshProUGUI)
   - Position at top-left or desired location
   - Assign to RaceHUD.timerDisplay

2. Speed Display (TextMeshProUGUI)
   - Position at top-right or desired location
   - Assign to RaceHUD.speedDisplay

Step 3: Configure RaceFinishFlowManager
--------------------------------------
1. Create TextMeshProUGUI for "FINISHED" message
2. Assign to RaceFinishFlowManager.finishedMessageDisplay
3. Set delayBeforeResults (e.g., 3 seconds)
4. Assign ResultsScreen reference

Step 4: Configure ResultsScreen
------------------------------
1. Create Results Table Container (Panel or VerticalLayoutGroup)
   - Assign to ResultsScreen.resultsTableContainer

2. Create ResultRow Prefab:
   a) Create a UI Panel/Image for one result row
   b) Add these child TextMeshProUGUI elements:
      - Placement Text
      - Athlete Name Text
      - Finish Time Text
   c) Add an Image component for Nationality placeholder
   d) Add ResultRow component to the panel
   e) Assign the text and image fields
   f) Drag the prefab to Assets/Resources or preferred location
   g) Assign to ResultsScreen.resultRowPrefab

3. Create Action Buttons (at bottom of Results Screen):
   - "Play Again" Button → assign to playAgainButton
   - "Replay" Button → assign to replayButton
   - "Main Menu" Button → assign to mainMenuButton

Step 5: Verify References
-------------------------
1. RaceManager should be auto-found via FindAnyObjectByType
2. If not assigned, manually assign in inspector
3. All event subscriptions happen in Start()

===============================================
RACE FLOW VERIFICATION
===============================================

The complete flow should be:

1. Pre-Race: HUD invisible, timer reset
2. Starting Sequence: On Your Marks → Get Set → Go
3. Race Starts: EnterRunningState fires
   → RaceTimer.StartTimer() called
   → RaceHUD.ShowHUD() called
   → Timer and Speed display visible
4. Player Finishes: Reaches finish distance
   → RaceManager.OnPlayerFinished fired
   → RaceTimer.StopTimer() called
   → RaceFinishFlowManager plays sequence
   → "FINISHED" message appears and fades
   → Waits configurable delay (3s default)
5. Results: ResultsScreen displayed
   → Results populated from RaceManager.GetRaceResults()
   → Buttons available for next action

===============================================
EXTENSIBILITY
===============================================

RaceHUD can easily be extended with new elements:
- Wind display
- Position/ranking
- Pace information
- Other metrics

Simply add new fields and update UpdateDisplay() method.
System maintains loose coupling via events.

ResultsScreen can be extended to show:
- Reaction time information
- Split times
- Detailed statistics
- Replay data (when implemented)

ResultRow can be updated to show additional data.

===============================================
IMPORTANT NOTES
===============================================

1. Official Finish Time:
   - Stored in RaceTimer when player finishes
   - Accessible via RaceManager.GetPlayerFinishTime()
   - Single source of truth (no recalculation)

2. Race Results:
   - Populated from RaceManager.GetRaceResults()
   - Returns List<RaceResult> sorted by placement
   - Supports multiple athletes (future-proof)

3. Scene Loading:
   - "Play Again" reloads current scene
   - "Main Menu" loads "MainMenu" scene
   - Ensure scene names are correct

4. Timing:
   - Timer starts EXACTLY when athlete movement begins
   - No pre-race time counted
   - Accurate to milliseconds
